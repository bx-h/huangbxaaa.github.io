    类和对象 #

    一些规则 #

    在Java中任何有效的代码必须写到“类”中，最外层必须是一个类的定义。类体中应该是方法或者变量，不能是直接的Java语句。

    一个Java源文件中可以定义多个class.

    public的类可以没有，如果有的话，每个Java文件中只能有一个，并且类名要和文件名一致.

    封装的作用：保证内部结构安全；暴露简单，屏蔽复杂

    package和import机制 #

    为什么要使用package？

    包机制可以方便程序管理，将不同功能的类分别存放在不同的包下。

    package怎么用？

    package + 包名。如package com.bjpowernode.javase

    命名规范？

    公司域名倒序 + 项目名 + 模块名 + 功能名

    如何编译和运行？

    类名不再是：HelloWorld，而是：com.bjpowernode.javase.HelloWorld

    路径在：com/bjpowernode/javase/HelloWorld

    编译：（生成对应文件夹）

    构造方法 #

    类如果不提供构造方法 会自动给予无参构造方法。假如自己写了任何构造方法（有参或者无参），编译器将不会自动添加无参构造方法。因此，要写构造方法就要把有参和无参都写了.

    构造方法格式: public className(parameters list)

    构造方法支持重载

    抽象类也属于引用数据类型。

    配合多态，可以实现面向抽象编程，降低程序的耦合度，提高程序的扩展力。

public class AbstractTest {
    public static void main(String[] args) {
        Animal a = new Cat();
        Animal b = new Dog();
        a.move();
        System.out.println("--------------");
        b.move();
        /*
        编译时，move方法指向的是Animal中的move方法。
        运行时，由于地址指向的是类Cat/Dog所在的堆内存，因此调用的是Cat/Dog的move方法
        */


    }
}

abstract class Animal {
    public abstract void move();
}

class Cat extends Animal {
    public void move() {
        System.out.println("Cat moving");
    }
}

class Dog extends Animal {
    public void move() {
        System.out.println("Dog moving");
    }
}

interface MyMath {

    1. 接口是一种引用数据类型
    2. 接口是完全抽象的
    3. 接口支持多继承接口中只包括两部分：
      1. 常量（public static final可以省略）
      2. 抽象方法（public abstract修饰符可以省略）
    4. 接口中所有的元素都是public
    5. 抽象方法不能带有方法体
    6. 有了接口就有了可插拔，扩展力很强

    类与类之间叫继承；类与接口之间叫做实现（采用implements关键字）。

    实现接口时，必须将接口中所有的方法全部实现。

    进行继承或者实现时，子类的访问权限不能比父类更低。

    int sum(int a, int b);
    int sub(int a, int b); //编译器会自动加上 public abstract

class MyMathImp implements MyMath {
    /*
    这个方法报错：理由：正在尝试分配更低的访问权限，应该加上public
    int sum(int a, int b) {
        return a + b
    }
    */
    public int sum(int a, int b) {}
    
    public int sub(int a, int b) {}
}

    public class AnimalTest {
    public static void main(String[] args) {
        Flyable f1 = new NewCat(12);
        f1.fly();
        Flyable f2 = new NewDog(13);
        f2.fly();
    }
}

class NewAnimal {
    int age;
    public NewAnimal(int age) {
        this.age = age;
    }
}

interface Flyable {
    void fly();
}

class NewCat extends NewAnimal implements Flyable {
    public NewCat(int age) {
        super(age);
    }

    @Override
    public void fly() {
        System.out.println("cat is flying");
    }
}

class NewDog extends NewAnimal implements Flyable {
    public NewDog(int age) {
        super(age);
    }

    @Override
    public void fly() {
        System.out.println("dog is flying");
    }
}

    接口相当于可插拔的。

    接口如何实现解耦：分离开调用者和实现者。

    注意：Java里面，不区分大小写。类名，接口名大小写不敏感。

    抽象类与接口的区别：

    抽象类是半抽象的。接口是完全抽象的。

    抽象类中有构造方法。接口中没有构造方法。

    接口和接口之间支持多继承。类和类之间只能单继承。

    一个类可以同时实现多个接口。一个抽象类只能继承一个类。（单继承）

    接口中只允许出现常量和抽象方法。

    以后接口使用的比抽象类多，一般抽象类使用的还是少。接口一般都是对“行为”的抽象。

    方法 #

    重载：方法名相同，参数不同。完成功能相似的方法。只有返回值不同不叫重载，编译器会判定为方法重复。

    参数传递，在栈中的基本变量进行复制，在堆中的引用变量进行地址传递.

    get和set方法都不带有static，都是实例方法。因为操作的都是对象的属性。

    抽象方法 #

    没有实现的方法，没有方法体的方法。

    特点：

    1. 没有方法体，以分号结尾
    2. 修饰符列表中有abstract关键字

    抽象类中可以没有抽象方法。抽象方法必须放在抽象类中。

    因此，子类（非抽象类）继承父类（抽象类），就必须对抽象方法进行实现/覆盖/重写。否则子类会继承父类的抽象方法而出错。

    判断题：没有方法体的方法都是抽象方法？（错误，Object类源码中就有很多方法没有方法体，底层调用的是C++动态链接库）

    变量 #

    引用数据类型：所有的类，String；

    基本数据类型：byte，short，int，long，float，double，char，boolean

    类中的实例变量（非静态）必须通过实例化后才能访问，否则不存在。

    对象的实例变量（成员变量）会附初始值0；方法中的局部变量则不会。

    私有属性变量，在static函数中不能访问。

    常量 #

    static final修饰的称为常量，常量名全部大写，每个单词采用下划线链接。

    常量和静态变量都存储在方法区，并且都在类加载时初始化。

    内存与机器 #

    编译器会自动寻找依赖文件进行编译。

    凡是通过new创建的对象，都存在堆中。地址付给局部变量。这种保存了地址的变量叫做引用。当一个对象没有引用指向，会自动被当做垃圾回收走

    修饰符 #

    final #

    final表示不可变的。final修饰的方法不能被覆盖，被重写，被继承。

    final修饰的变量只能赋一次值，一旦赋值不可改。

    final修饰的实例变量，系统不会赋给它默认值，要求程序员手动赋值，可以直接定义时赋值或者在构造方法中赋值。（不背锅）。假如没有final则会赋默认值0。
